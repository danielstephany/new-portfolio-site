The portfolio

I have worked on quite a few projects since I started web development a few years ago, here are some that I have selected to share. Some of these are client work and some are just personal projects created while on the journy to learn something new.

  • Evercord

    Evercord

    Evercord is a company that banks newborns cord blood so that they can use the stem cells it contains to treat disease in the future. The plan for the Evercord site was to build it out and launch statically first to meet the deadline, and then hand it over to a backend developer to put it in a CMS post launch. Since the Evercord site was supposed to be on such a tight deadline and we had no content that was going to be approved until later in the project we used a strategy of giving the client a design guide with content sections and site components laid out so they could get an idea of the building blocks that pages would consist of. This allowed us to be very flexible when building out pages.

    This was a project that had changes coming in from the client constantly, so I learned a lot about keeping my code organized, and structured in a way that made it easy to read and jump into.

    Below are some of the parts I worked on:

    • I was involved in coding out the functionality and styling for some aspect of every page on this site.
    • I build out all of the pages in their enrollment flow which consisted of lots of forms with sections that toggled open or closed depending on selected inputs.
    • On the homepage I added the full screen video banner with a image fallback.
    • For video modals I modified a modal plugin that I built for use with youtube videos to instead work with Wistia embeds. I also added the ability to add another video that gets displayed below the currently playing video with a title and a description.
    • The header for this site was built using a fully responsive header plugin I built previously and modified for this project. It was animated using CSS3 animations to draw attention to the get started button.

    On this site there are several different sections that show content depending on which icon is selected these were all built using the slide select plugin that I built for this project. I am proud of this plugin because I built it to be very versatile in the way that it can be styled and used. This single plugin was used for the functionality of 3 very differently styled elements on this site, the infographic that allows the user to toggle through diseases and view the associated part of the body, a circular infographic that lets the user toggle between the information shown in the center, and a row of icons that lets a user toggle through associated information. I built out the pricing and get started pages that were built to work with dynamically generated content pulled from and api. A part of this page that I particularly enjoyed working on was building the floating order summary that is shown on desktop. This is an element that becomes fixed and floats down the screen with the user until they get to the bottom of the price selection section, then the element becomes positioned absolute and sticks to the bottom of the pricing section.

  • Genepeeks

    Genepeeks

    Genepeeks is a genetic testing company that helps parents identify if their child be at risk of disease. My role in this project was to work remotely with the genepeeks development team to build out the front end of the report page that test results would be shown on and to add a new homepage, navigation to their website.

    Below are some of the parts I worked on:

    • The nav on this site I coded from scratch, it is able to have navigation 3 layers deep, and is fully responsive.
    • I built out the 1000 infographic on the page which is animated to go from 0 to 1000 when it enters the screen.
    • The video modals on this site are a plugin that i built which allows a developer to add a data-video attribute with a reference to a youtube video to any element on the page which will then make that element into a click target for a video modal.
    • On the report page I coded a menu that slides down the page with the user and highlights menu items depending on the user's location on the page.

    The last thing that I worked on for this project was going the pages that were previously on the genepeeks site and updating the styles so that every page on the site was fully responsive down to mobile sizes. I was the only developer on this task and was given the freedom to take the current pages and tweak the layout and design so that every page on the site always presented a great experience to the user now matter what screen they viewed the site on. This task was very rewarding I really enjoyed creating a better user experience for all screen sizes.

  • Specific

    Specific

    Specific is a company trying to change the way blood infection is treated by offering fast cost-effective solutions for detection. A challenging part of this project was definitely the deadline we had about a week and a half to build out this site and I don't even think we had final designs.

    Below are some of the parts I worked on:

    • I built the sliding content sections Seen on the how it works overview for desktop using the greensock animation library along with scrollmagic.
    • I styled and built out the functionality of the global header, footer, and secondary navigation.
    • The form component in the partner with us section was an interesting challenge. Their are 5 choices under the partner with us section that all lead to the same page, but the form element and the text above it and the select input within it is changed based on the hash in the link. Also if you are already on the page and the user chooses a different partner type from the menu you stay on the same page but the form is updated to reflect the different type.

  • Invivoscribe

    Invivoscribe

    I worked on the front-end of the Invivoscribe site in a team of 4 developers. Invivoscribe is a company that develops and manufactures diagnostic products and reagents for leukemias and lymphomas. While working on this project I was responsible for building out the front-end for the e commerce portion of the site based off of wireframes for desktop. This was a good challenge when it came to scaling down to mobile sizes. This site contained a lot of information that needed to be displayed in a way that wouldn’t overwhelm the user, that required us to have a lot of hidden content that would be revealed in modals, accordions or expanding content sections. We built the Invivoscribe site to be compatible on all modern browsers and with ie support back through version 8.
